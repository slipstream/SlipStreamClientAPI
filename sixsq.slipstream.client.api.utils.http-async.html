sixsq.slipstream.client.api.utils.http-async

An asynchronous wrapper arount an HTTP library for simple CRUD actions. All functions return a core.async channel that will contain the HTTP response.

The functions mirror those in the synchronous wrapper and generally take the same arguments. An difference is that exceptions are not thrown on HTTP error response codes and are instead pushed into the channel.
